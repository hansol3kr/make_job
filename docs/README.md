# (가칭) 지금인력 — 실시간 인력 아웃소싱 플랫폼

> **한 줄 정의:** "지원자"가 아니라 **확정된, 신뢰할 수 있는 사람**을 실시간으로 매칭하고, 안 되면 자동으로 대체 인력을 채워주는(백필) 양방향 신뢰 기반 인력 마켓플레이스.

이 문서는 시장 조사 → 차별점 → 설계 → 구현으로 이어지는 프로젝트의 **설계 기준 문서(source of truth)** 입니다. 코드보다 먼저 읽으세요.

## 1. 시장 요약 (왜 지금, 왜 우리)

- 한국 온디맨드 인력 시장은 3세대까지 왔지만 **진짜 실시간 디스패치는 사실상 급구(니더) 하나뿐**. 나머지(당근알바·알바몬·알바천국·벼룩시장·일다오)는 전부 **"지원자까지"** 만 준다 → "확정된 사람"까지는 수동 채팅.
- 시장 **최대 통증은 노쇼/직전취소**. (인용된 "자영업자 82% 노쇼" 통계는 출처가 약해 **검증 대상**이나, 통증 자체는 모든 리뷰에서 반복 확인됨.)
- 인접 마켓플레이스(숨고·크몽)는 **피해야 할 가격/신뢰 안티패턴**을 보여줌: 숨고의 견적당 과금(성사 무관 ₩2~7천), 크몽의 양측 이중 수수료(~20%).
- 글로벌 레퍼런스는 **Instawork**(연속 ML 매칭, 높은 충원율, 낮은 노쇼, 신뢰도 점수/Top-Pro 티어)와 국내 신뢰 벤치마크 **해주세요**(ID+계좌+사진+테스트 인증, 선결제 에스크로, 대칭 노쇼 페널티 명문화).

> **화이트 스페이스 = 아무도 동시에 잡지 않은 교집합:** ① SLA 기반 "확정/백필 보장" ② 이빨 있는 양방향 책임 ③ 투명·설명가능한 공정성 ④ 기능으로서의 컴플라이언스. 이걸 **1개 밀집 지역 + 1개 고빈도 버티컬**에서 유동성부터 확보하며 시작.

## 2. 우리의 차별점 (6개 축)

1. **"공고"가 아니라 "확정된 사람"을 판다** — 연속 매칭 + e-계약 잠금 + 취소 즉시 사전심사된 대체자 자동 호출(백필). 광고/구직 보드는 정산·백필 레이어가 없어 구조적으로 못 함.
2. **급구를 속도가 아니라 신뢰·공정성·밀도로 이긴다** — 더블블라인드 동시공개 평점, 이빨 있는 양방향 취소이력, 설명가능한 랭킹("왜 이 제안/이 급여/이 거리"), **거절해도 불이익 없음**. 그리고 급구가 약한 비수도권/특정 버티컬을 먼저 채운다.
3. **GPS 기반 대칭 노쇼 책임제** — 시장 1위 통증을 정면 공략. 유예창 → 25/50/100% → 보증금 몰수, 양측 신뢰도 점수 연동, 상대 귀책 시 자동 면제 + 이의신청.
4. **기능으로서의 컴플라이언스가 곧 해자** — 단기 근무는 기본을 **일용근로소득**으로(가짜 3.3% 아님), 근무 전 **e-근로계약서 자동 발급**, 산재/고용보험 특례 자동화. 중립 직업소개 구조로 타다 리스크 차단.
5. **투명한 성사 기반 단측 과금** — 완료·에스크로 정산된 건에만 과금, 총액 선공개, 라이선스 PG로 당일 정산.
6. **낯선 사람 간 대면 노동을 위한 신뢰 번들** — 본인확인기관 ID(주민번호 원문 미보관) + 계좌 소유 확인 + 완료 시까지 에스크로 + 안심번호 + 실시간 위치공유 + 원터치 SOS를 "Protected by [App]"로 패키징.

## 3. 30년차의 직언 — 이 사업의 진짜 리스크와 우리의 방어 (반드시 읽기)

리서치 비평 에이전트가 냉정하게 지적한 것들. 여기에 정직하게 답하지 못하면 앱은 실패합니다.

| 하드 트루스 | 왜 위험한가 | 우리의 de-risk 설계 |
|---|---|---|
| **① 트릴레마(3중 딜레마)** | "확정 보장"을 지키려면 매칭을 강하게 통제해야 하는데, 강하게 통제할수록 (a) 플랫폼=사용자(타다 근로자성) 리스크가 커지고 (b) 급구가 그대로 베낄 기능 목록이 됨. 통제를 풀면 충원을 보장 못 함. | **급여결정·수락·징계 권한을 요청자(사용자)에게 남긴다.** 우리는 "소개"만. 거절 불이익 없음(=근로자성 완화 + 공정성 차별점). 보장은 "강제 배차"가 아니라 "자동 백필 + 실패 시 수수료 0"의 **정직한 SLA**로. |
| **② 콜드스타트 유동성** | 실시간 "즉시 확정"은 **놀고 있는 대기 공급**이 있어야 성립. 씨드 시장엔 그게 없음. Handy가 이걸로 망함. "지역 하나 씨딩"은 문제 재진술이지 해법이 아님. | **하드 SLA를 초기부터 걸지 않는다.** 1개 밀집 지역 + 1개 고빈도 반복 버티컬에 **양측 동시 씨딩**(공급 인센티브 예산 포함) → 밀도 확보 후 확장. 첫 100 근로자/100 업주 확보 플랜을 GTM 문서에 명시. |
| **③ 유닛이코노믹스 & 요율 상한** | 유료직업소개 요율은 법적 상한(사용자 ≤30% 1회성, 건설일용 10%, 근로자 ≤1%, 성사 후에만). 저단가(₩8~15만) 건에 인증+보험+안심번호+PG+에스크로 비용 스택이 안 맞음. | 수수료만으로 못 버틴다고 전제. **결제 처리·프리미엄 노출·고빈도 업주 구독·건당 보험**으로 보완. 신뢰 스택 비용은 티어링/배치로 비례화. (전용 문서에서 수치화) |
| **④ 디스인터미디에이션(이탈)** | 고빈도 반복 버티컬일수록, 업주가 좋은 근로자 1명 찾으면 **다음부턴 앱 밖 직접 고용**. 마켓의 최대 구조적 누수. | 온플랫폼 전용 혜택으로 락인: 신뢰도 이력·에스크로·보험·원터치 재예약("이 분 다시 부르기")·로열티 크레딧이 **앱 안에서만** 유효. |
| **⑤ 검증 안 된 통계** | "82% 노쇼", "3초 매칭", "노쇼 <2%" 등은 마케팅/자기보고 수치. 목표치로 그대로 쓰면 위험. | 방향성 신호로만 사용. **파일럿에서 우리 실측 KPI**(충원율, 확정 후 노쇼율, 확정까지 시간)로 가설 검증 후 스케일. |
