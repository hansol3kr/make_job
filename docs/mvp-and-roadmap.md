# MVP 범위 · 로드맵 · 법률 체크리스트

## 1. MVP 범위 (de-risk 반영)

플랫폼은 **전방위(카테고리 구조)** 로 짓되, **GTM은 1개 버티컬 + 1개 밀집 지역**으로 좁혀 유동성부터 확보한다.

### Must (MVP 필수)
- 원터치 요청 → **연속 매칭 → "확정된 근로자"** (지원자 아님). 반경·ETA·신뢰도·수락예측으로 랭킹, 30~60s 수락 타이머, 푸시로 백그라운드 도달.
- **자동 백필** — 취소/노쇼 시 다음 사전심사 후보에게 즉시 재오퍼. (제품의 핵심 약속)
- 양방향 **본인 인증 + 신뢰도 점수** — 본인확인기관 ID(주민번호 원문 미보관), 계좌 소유확인, 롤링 신뢰도.
- **더블블라인드 동시공개 평점** — 완료·정산 건만, 세부점수, 공개 즉시 잠금.
- **대칭 노쇼/취소 페널티** — 유예창 + 티어 + 신뢰도 연동 + 자동 면제 + 이의신청.
- **에스크로 + 당일 정산** — 라이선스 PG 경유, 승인→에스크로→완료 릴리스. 자금 우리 계좌 미보관.
- **컴플라이언스 인 플로우** — 근무 전 e-근로계약서 자동, 단기는 **일용근로소득** 기본.
- **안심번호 + 인앱 로그 채팅** — 전화번호 미노출, 분쟁 증거.
- **정직한 SLA** — "평균 N분 확정, 못 채우면 자동 백필, 그래도 실패 시 수수료 0". 강제 배차·거절 불이익 없음.

### Should (있으면 강함)
- 신뢰도 티어(Top-Pro 유사): 우선 오퍼·서지 급여·즉시 정산.
- 보증금/몰수(에스크로 레일로 자동).
- 실시간 위치 공유 + 원터치 SOS(양방향).
- 서지/동적 급여(비선호·긴급 슬롯 충원, 총액 선공개).
- 시간박스 분쟁 플로우(48~72h 증거 SLA + 무응답 자동규칙).
- 설명가능성 UI("왜 이 오퍼/급여/거리") + "지금은 아님"과 "거절" 분리.

### Later (후속)
- 건당 임베디드 보험("Protected by [App]") + 수익 공유.
- 블루칼라 현장 버티컬(전기/배관/목공 검증 배지, 투명 정가).
- 불규칙 소득자 금융툴(선지급/EWA, 소득증빙·신용형성).
- EOR/W-2형 프리미엄 티어(근로자성 리스크 흡수 업셀).
- AI 운영(분쟁 트리아지, 리뷰 어뷰징 탐지, 챗봇 공고작성).
- 롤링 이분매칭(Hungarian) + ML ETA 고도화, 지역 확장.

## 2. GTM 씨딩 (콜드스타트 방어의 핵심)
- **1개 밀집 지역 + 1개 고빈도 반복 버티컬**을 골라 **양측 동시 씨딩**.
  - 후보 버티컬: (A) F&B·편의점·리테일 **매장 단기/대타**, (B) **물류/창고**, (C) 생활 심부름·용역, (D) 블루칼라 현장.
- 첫 100 근로자 / 첫 100 업주 확보 플랜 + 공급 인센티브(대기 보상) 예산을 명시적으로 편성.
- **이탈(디스인터미디에이션) 방어**: 신뢰도 이력·에스크로·보험·"이 분 다시 부르기"·로열티 크레딧을 온플랫폼 전용으로.
- 파일럿 KPI로 가설 검증 후 확장: **충원율 / 확정까지 시간 / 확정 후 노쇼율 / 재사용률(리텐션)**.

## 3. 구현 로드맵 (마일스톤)

| 단계 | 목표 | 산출물 |
|---|---|---|
| **M0 — 기반** | 워크스페이스·Supabase·스키마·인증·앱 스켈레톤 | Flutter(Android) 앱 골격, Supabase 프로젝트, 마이그레이션(스키마+RLS), 휴대폰 OTP 로그인, 디자인 시스템 토큰 |
| **M1 — 코어 루프(해피패스)** | 요청→매칭→오퍼/수락→배정→체크인/아웃→완료 | PostGIS 후보조회 + Edge Function 매칭 + Realtime 오퍼, 지도 UI, 결제는 목(mock) |
| **M2 — 신뢰** | 인증·평점·신뢰도·페널티·백필 | ID/계좌 인증, 더블블라인드 평점, 신뢰도 점수, 취소/페널티 규칙, 자동 백필 |
| **M3 — 결제·컴플라이언스** | 에스크로·당일정산·계약 | PG 에스크로 연동, 당일 정산, e-근로계약서 생성, 일용근로소득 처리 |
| **M4 — 안전·완성** | 안심번호·SOS·분쟁·성능·iOS | 안심번호, 위치공유/SOS, 분쟁 플로우, 알림, 성능 튜닝, **iOS 빌드(Mac)** |
| **파일럿** | 1지역·1버티컬 클로즈드 베타 | 실측 KPI 수집·검증 |

> **환경 메모(구현 시):** 이 리눅스 환경에는 Flutter/Android SDK/JDK가 아직 없음 → M0에서 설치 필요(수 GB). 에뮬레이터가 어려우면 **Flutter Web 프리뷰**로 UI를 먼저 "확인"하고, APK 빌드로 실기기 검증. iOS는 Mac 필요.

## 4. 법률 · 컴플라이언스 체크리스트 (한국)

> ⚠️ 실제 사업화 전 **노무·전자금융·개인정보 전문 변호사 검토 필수.** 아래는 설계 반영용 요약.

- [ ] **유료직업소개사업 등록** (자본금 5천만, 자격 임원, 사무실 ≥10㎡). 요율 상한: 사용자 ≤30%(1회성)·건설일용 10%, 근로자 ≤1%, **성사 후에만** 징수 → 수수료 수익 상한 → 결제/보험/구독으로 보완.
- [ ] **중립 "직업소개" 포지션 고정**: 요청자가 사용자(=고용주)로서 급여 결정·업무 지휘·근로계약 당사자. 플랫폼은 소개만. **"인력 파견/공급"으로 마케팅·설계 금지** (근로자공급·위장도급·불법파견은 형사처벌 + 직접고용 의무 리스크).
- [ ] **플랫폼=사용자(타다) 리스크 완화**: 알고리즘 통제(급여결정·수락강제·징계) 최소화. 거절 불이익 없음, 강제 배차 없음. `근로자 추정제` 대비.
- [ ] **전자금융거래법**: 정산/에스크로는 **라이선스 PG 경유**, 자금 우리 계좌 미보관. (2026.12.17 100% 안전관리 의무 시행 대비 — 자체 전금업 라이선스는 자본 부담 큼.)
- [ ] **세무(가짜 3.3 금지)**: 단기 1일 근무는 **일용근로소득**(일 ~18.7만원까지 원천징수 미미) 기본. 근로내용확인신고, 간이지급명세서 정합성.
- [ ] **4대보험/산재**: 산재(시간 무관) + 고용보험 노무제공자 특례 자동 안내/신고(후속 자동화).
- [ ] **위치기반서비스사업 신고**(방통위) + **위치정보 이용 동의**. 정밀 위치 짧은 보존.
- [ ] **개인정보보호법**: 주민번호 원문 미수집(대체수단)·암호화, 최소수집. 위반 시 과징금(관련 매출 최대 3%).
- [ ] **직전취소 → 휴업수당(근기법 §46, 평균임금 ~70%)** 노출 가능성(플랫폼=사용자 리스크 현실화 시). 페널티 설계와 책임 귀속 명확화.
- [ ] **청소년/유해업종 스크리닝** (재가·야간 근무 타겟 시 특히).

## 5. 미결정 사항 (사용자 확인 필요)
1. **프레임워크 확정**: Flutter(추천) vs React Native/Expo.
2. **씨드 버티컬/지역**: 어느 버티컬을 먼저 씨딩할지 (플랫폼은 전방위지만 시작점 1개).
3. 앱 이름/브랜드 (현재 가칭 "지금인력").
4. PG 파트너 선호 (PortOne vs 토스페이먼츠).
