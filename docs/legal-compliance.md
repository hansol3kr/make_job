# 법적 컴플라이언스 — 리스크·동의·급여 (판사 관점 분석)

> ⚠️ **면책:** 본 문서는 한국 현행법 지식에 기반한 정밀 분석이며 **정식 법률자문이 아니다.**
> 실제 자금·근로자를 다루기 전 **노무사·변호사 검토는 필수**다. 아래는 리스크를 최소화하는 설계.

## 1. 앱의 법적 포지션 (생명선)

이 앱은 **직업안정법상 "유료직업소개사업"** + **전자상거래법상 "통신판매중개자"** 로 포지셔닝한다.
**플랫폼은 사용자(사업주)가 아니다.** 근로관계는 **요청자(업주)↔근로자** 사이에 직접 성립하고,
플랫폼은 **중개**만 한다. 이 경계가 무너지면 근로기준법상 사용자 책임(임금·4대보험·퇴직금·해고제한)이
플랫폼으로 전가된다.

### 회피해야 할 3대 함정
| 리스크 | 법 | 방어 |
|---|---|---|
| **근로자파견** | 파견법 | 플랫폼이 근로자를 지휘·파견하면 무허가 파견(형사처벌). → **지휘·감독·급여결정·수락권을 업주에 귀속** 명문화 |
| **근로자공급** | 직업안정법 | "확정 배차"가 공급으로 보일 위험. → **거절 불이익 없음·강제배차 없음**(이미 SLA), 근로자 자유수락 |
| **플랫폼=사용자** | 근로기준법 | 급여를 플랫폼이 "지급"하면 사용자 오인. → **급여는 업주→근로자, 플랫폼은 지급대행/에스크로 중개** |

|---|---|---|---|
| 1 | **서비스 이용약관** (플랫폼=중개자, 비사용자 명문화) | 전자상거래법 | ✅ |
| 2 | **개인정보 수집·이용** (이름·전화·위치·계좌·프로필) | 개인정보법 §15 | ✅ |
| 3 | **개인정보 제3자 제공** (매칭 시 상대방에 제공) | 개인정보법 §17 | ✅ |
| 4 | **위치기반서비스 이용약관 + 개인위치정보 수집·이용** | 위치정보법 §18·19 | ✅ |

## 4. 급여 지불 설계 (권장) — §5 상세

핵심: **플랫폼이 자금을 보유하면 전자금융업(선불/에스크로) 등록 대상.** → **PG 에스크로로 회피.**

```
근무 전:  업주 → [PG 에스크로 선결제] 급여+수수료   (자금은 PG 보유, 플랫폼 미보유)
근무 완료(체크아웃 확인) → PG 정산:  근로자에 급여 지급 + 플랫폼 수수료 차감
매칭 실패/노쇼:  에스크로 환불(자동 백필 실패 시 수수료 0 — SLA)
```

- **자금 미보유** → 전금법 회피 (PortOne/토스페이먼츠 에스크로).
- **급여 = 업주→근로자 직접**(플랫폼은 지급대행 표시) → 사용자 오인 회피.
- **원천징수**: 일용근로소득세 원천징수의무자 = **업주**. 플랫폼은 계산·신고 **대행 옵션** 제공(가짜 3.3% 사업소득 금지 — 일용근로소득으로).
- **산재보험**: 일용근로자 산재 적용, 업주 부담. 플랫폼 노무제공자 특례 여부 확인.
- **최저임금 검증**: 요청 급여 ÷ 근무시간 ≥ 최저임금 미달 시 등록 차단(구현 권장).
- **정산 투명성**: 근로자에 급여명세(항목·공제) 제공(근로기준법 §48 임금명세서 교부 의무).
